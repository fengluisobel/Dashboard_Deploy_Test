## 📖 核心功能使用指南

### 1️⃣ 翻转卡片系统 - 让指标会"讲故事"

**位置**: 所有仪表板顶部的核心指标区域

**如何使用**:
1. 将鼠标**悬停**在任意指标卡片上
2. 卡片会以3D效果翻转180度(0.6秒平滑动画)
3. **背面显示**:
   - 📐 **计算公式** - 这个指标是怎么算出来的
   - 📈 **数据明细** - 分子分母的原始数值
   - 🎯 **基准参考** - 优秀/良好/需改进的标准

**示例 - 关键战略岗位按时达成率**:
```
┌─────────────────────────────┐
│ 正面 (默认显示)             │
├─────────────────────────────┤
│ 关键战略岗位按时达成率      │
│        85.2%                │
│   ▲ 0.2% vs 目标           │
└─────────────────────────────┘

        ↓ 悬停翻转 ↓

┌─────────────────────────────┐
│ 背面 (悬停显示)             │
├─────────────────────────────┤
│ 📊 关键战略岗位按时达成率   │
│                             │
│ 📐 计算公式:                │
│ 按时入职P0级人员数 /        │
│ P0级招聘计划总数 × 100%     │
│                             │
│ 📈 数据明细:                │
│ 按时入职P0级人员: 34人      │
│ P0级招聘计划总数: 40人      │
│                             │
│ 🎯 基准参考:                │
│ 优秀: >85% | 良好: 75-85%  │
│ 需改进: <75%                │
└─────────────────────────────┘
```

**三种主题**:
- 🟣 **HRVP主题**: 蓝紫色,战略视角
- 🟢 **HRD主题**: 绿色,管理视角
- 🟠 **HR主题**: 橙色,执行视角

### 2️⃣ 硅碳比分析 - AI与HR的协同效能仪表盘

**位置**: HRVP战略驾驶舱 → 滚动到"图表7: 硅碳比分析"

**核心概念**:
```
硅碳比 = AI等效人力 / HR团队人数

AI等效人力 = HR团队人数 × AI平均承接率%
```

**图表说明**:
- **蓝色柱状图**: 各部门的硅碳比
  - 数值越高 = AI承接越充分
  - 标杆值: 0.7 (优秀), 0.4-0.7 (良好), <0.4 (待提升)

- **绿色折线图**: HR负载释放率(%)
  - 表示AI接手后,HR从低价值工作中释放出来的比例
  - 越高越好,目标 >60%

- **灰色虚线**: 平均硅碳比基准线

**如何解读**:

| 硅碳比 | HR负载 | 含义 | 建议行动 |
|--------|--------|------|----------|
| **< 0.4** | > 6人/月 | 🔴 AI承接不足,HR压力大 | 优先增加AI算力(低成本5万/月) |
| **0.4-0.7** | 4-6人/月 | ⚠️ 有提升空间 | 优化AI模型准确率 |
| **> 0.7** | < 4人/月 | ✅ 资源配置优秀 | 保持当前策略 |
| **> 0.6** | > 8人/月 | 🔴 AI已充分,但任务量大 | 增加碳基HR人力 |

**详细数据表**:
图表下方有7列数据:
- 部门、HR人数、AI等效人力、硅碳比
- 负载释放率(%)、效率提升(%)、协同健康度

**智能决策建议**:
系统会根据数据自动生成3个维度的建议:
1. 平均HR月负载(偏高/健康)
2. AI平均承接率(优秀/待提升)
3. 协同效率提升(vs 纯人工模式)

### 3️⃣ 校招质量九宫格 - 未来人才的战略雷达

**位置**: HRVP战略驾驶舱 → 滚动到"图表8: 校招人才质量分析"

**质量矩阵图**:
```
  面试分(软技能)
     100 ↑
         │
      85 │  左上象限        │  右上象限 ✅
         │  沟通强技术弱    │  双高人才(最优)
         │  蓝色气泡        │  绿色气泡
    -----├─────────────────┼─────────────────
      80 │                 │ ← 高潜线
         │  左下象限        │  右下象限
      65 │  双低 ⚠️         │  技术强沟通弱
         │  红色气泡        │  橙色气泡
         │_________________│_________→ 笔试分(硬技能)
        65      80 ↑       85        100
                 高潜线
```

**4个象限解读**:
- **右上 - 双高人才 (绿色)**: 笔试≥80 且 面试≥80
  - 💎 **最优区域**,未来3-5年核心战斗力
  - 目标: 集中60%+的Offer在此象限

- **右下 - 技术强沟通弱 (橙色)**: 笔试≥80 但 面试<80
  - 🔧 技术岗可用,需加强软技能培养

- **左上 - 沟通强技术弱 (蓝色)**: 笔试<80 但 面试≥80
  - 🎯 适合商务、运营等非技术岗

- **左下 - 双低 (红色)**: 笔试<80 且 面试<80
  - ⚠️ 需审慎考虑是否发Offer

**顶部KPI卡片**:
- **校招综合质量**: 基于笔试、面试、S级占比、签约率的加权得分
- **S级人才占比**: 目标15%+
- **总体签约率**: 健康线70%
- **S级签约率**: 关键指标,反映顶尖人才争夺战胜率

**院校来源分析**:
- 顶级院校占比 (C9联盟 + 海外QS50)
- 985/211占比 (中坚力量)
- 海外院校占比 (国际化程度)

**S级人才流失原因**:
柱状图展示4大拒签原因:
- 🔴 **薪资不达预期** - 占比最高,建议建立SSP机制
- 🟠 **竞对(BAT)截胡** - 需加强雇主品牌和提前布局
- 🔵 **工作地点不符** - 可考虑远程/灵活办公
- ⚪ **其他原因** - 个案分析

**战略决策建议**:
1. **S级人才专项**: 设立SSP特殊薪资包,签约率目标 42% → 65%
2. **提前布局**: 大二暑期实习计划,锁定潜力人才
3. **差异化竞争**: 避开BAT主战场,聚焦新兴赛道人才

### 3️⃣ HR负载与效能趋势图

**位置**: 滚动到 **"5️⃣ HR 招聘负载与效能趋势 (碳硅协同视角)"**

**图表说明**:
- **蓝色柱状图**: HR人均招聘负载 (人/HR)
  - 表示每个HR平均承担的招聘人数
  - 越高表示负载越重

- **绿色折线图**: 招聘转化率 (%)
  - 表示从简历筛选到最终入职的转化效率
  - 越高表示效率越好

- **橙色虚线**: 负载警戒线
  - 自动计算 = 平均负载 × 1.2
  - 超过此线表示负载过高,需要关注

**如何解读**:
```
理想状态: 负载稳定 + 转化率提升
  ║ 负载 ↓ ↓ ↓     转化率 ↗ ↗ ↗
  ║  ▌ ▌ ▌              ╱
  ║  ▌ ▌ ▌            ╱
  ╚════════════════════
   1月 2月 3月

警示状态: 负载上升 + 转化率下降
  ║ 负载 ↗ ↗ ↗     转化率 ↘ ↘ ↘
  ║      ▌ ▌           ╲
  ║    ▌ ▌ ▌            ╲
  ╚════════════════════
   1月 2月 3月
```

**洞察分析**:
图表下方有两列智能分析:
- **左侧**: 负载分析
  - 当前人均负载
  - 负载趋势 (上升/下降)
  - 资源状态 (🔴过高/⚠️偏高/✅健康)

- **右侧**: 效率分析
  - 当前转化率
  - 平均转化率
  - 优化建议 (根据状态自动生成)

### 4️⃣ HR资源配置与缺口分析

**位置**: 滚动到 **"6️⃣ HR 资源配置与缺口分析"**

**图表说明**:
- **散点图**: 每个气泡代表一个部门
- **X轴**: HR人均负载 (招聘人数)
- **Y轴**: 关键战略岗位达成率 (%)
- **气泡大小**: 简历筛选量 (越大表示工作量越大)
- **颜色编码**:
  - 🔴 **红色** = 需增员 (负载 > 平均×1.3)
  - ⚠️ **橙色** = 需关注 (负载 > 平均×1.1)
  - ✅ **绿色** = 健康 (负载正常)

**四象限解读**:
```
  高达成率
      │
  左上象限 │ 右上象限
  (最优)   │ (优秀但负载高)
  ────────┼────────
  左下象限 │ 右下象限
  (效能问题)│ (严重问题!)
      │
  低达成率
      低负载     高负载
```

- **左上象限** (✅ 最优): 低负载+高达成 → 值得推广经验
- **右上象限** (⚠️ 关注): 高负载+高达成 → 关注可持续性
- **左下象限** (⚠️ 分析): 低负载+低达成 → 非负载问题,需分析根因
- **右下象限** (🔴 紧急): 高负载+低达成 → 需立即增援或优化

## 💡 使用技巧

### 技巧1: 快速发现问题部门
1. 点击 **[近3个月]** 快捷筛选
2. 滚动到 **图表6: HR资源配置与缺口分析**
3. 重点关注 **右下象限的红色气泡**
4. 这些是高负载+低达成的问题部门,需立即处理

### 技巧2: 验证指标计算
1. 在核心指标卡片区,看到某个异常值
2. **悬停该卡片** 查看背面
3. 检查 **计算公式** 和 **原始数据**
4. 确认是数据问题还是计算问题

### 技巧3: 对比不同时间段
1. 点击 **[近3个月]**,记录图表数据
2. 点击 **[近半年]**,观察变化
3. 对比 **负载趋势** 和 **资源缺口** 的变化
4. 判断问题是短期波动还是长期趋势

### 技巧4: 导出数据分析
1. 使用侧边栏的 **"📤 导出全量数据至Excel"** 按钮
2. 下载后可以在Excel中进一步分析
3. 文件包含3个工作表:
   - 环节负载效能数据
   - 部门产能释放数据
   - 岗位成本周期数据

## ❓ 常见问题

### Q1: 翻转卡片没有反应?
**A**: 确保鼠标完全悬停在卡片上,翻转动画0.6秒后生效

### Q2: 快捷筛选不生效?
**A**: 检查是否有自定义筛选冲突,点击"全部时间"重置后再试

### Q3: 图表显示空白?
**A**: 检查数据是否包含"简历筛选总数"列,已在data_generator_complete.py:291修复

### Q4: 负载警戒线位置不对?
**A**: 警戒线 = 平均负载×1.2,会根据数据自动计算

### Q5: 如何自定义颜色?
**A**: 修改 flip_card_system.py 中的主题色:
- HRVP主题: `#4A5FE8` (蓝紫)
- HRD主题: `#0D7C3A` (绿色)
- HR主题: `#C17A00` (橙色)
